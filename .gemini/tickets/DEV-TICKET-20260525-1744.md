# DEV-TICKET-20260525-1744: Reorganize Core Specification Directories

**Title**: Consolidate and Reorganize Root-Level Specification/Agent Directories
**Origin**: `/harness-chat` session (Trace ID: 2b24eeb0-10c3-46d8-9042-e360723fcecd)
**Status**: CLOSED

## Description
The current repository structure has multiple top-level directories for "Agentic SDD" metadata (`agents/`, `prompts/`, `specs/`, `tickets/`). This causes root-directory bloat and hides the distinction between "Service Specifications" and "System Harness Mandates." The user requested moving agents/skills to `.gemini/` to clean up the workspace.

## Identified Flaw
* Lack of a consolidated infrastructure/harness directory leads to fragmented path references and a high-clutter root environment, which may impact discoverability for new services.
* The user wants to be able to access the triggers like `/solve-tickets` in the cli as other gemini cli commands

## Proposed Action
1. **Consolidation**: Investigate moving `agents/`, `prompts/`, and potentially `tickets/` into a single parent directory (e.g., `harness/` or `meta/`).
2. **Path Update**: Perform a global search and replace for all path references in:
   - `GEMINI.md`
   - All `.md` files in `agents/` (Initial Context blocks)
   - `scripts/*.sh`
3. **GitOps Compliance**: Ensure `.gitignore` is updated so that these core mandates remain versioned even if moved to a dot-prefixed directory (if that path is chosen).
4. **CLI Validation**: Verify that the Gemini CLI can still locate the `system-context.md` and other mandates post-migration.

## Resolution Summary
- **Consolidation**: Successfully moved `agents/`, `prompts/`, and `tickets/` to `.gemini/`.
- **Path Updates**: Updated all path references in `GEMINI.md`, agent definitions (`.gemini/agents/*.md`), `specs/harness/telemetry-protocol.md`, and documentation.
- **GitOps**: Updated `.gitignore` to explicitly allow tracking of the new `.gemini/` subdirectories.
- **Verification**: Telemetry audit passed (`scripts/audit-telemetry.sh`). CLI triggers remain functional through `.gemini/agents/` discovery.
