# DEV-TICKET-20260525-1650.md: Centralize Ansible Infrastructure Logic

## Title
Refactor Harness to use a Centralized Ansible Directory

## Origin
Harness Chat Consultation (Trace ID: f7e6d5c4-b3a2-4123-8901-23456789abcd)

## Description
The current harness distributes Ansible playbooks across `scripts/` and `environments/` (e.g., `environments/prod/setup.yml`, `environments/staging/setup-staging.yml`). This leads to logic duplication for core tasks such as:
- Docker Engine installation.
- SSD/Storage provisioning.
- UFW and SSH security hardening.

This redundancy increases the risk of drift between the staging simulation and the physical bare-metal host.

## Proposed Action
1. Create a root `ansible/` directory.
2. Implement a `roles/` structure to encapsulate reusable logic:
    - `roles/docker`: Engine and Compose installation.
    - `roles/storage`: SSD formatting and fstab mounting.
    - `roles/security`: UFW rules and SSH hardening.
3. Refactor existing environment setup scripts to utilize these roles via a central `site.yml` or environment-specific playbooks in the `ansible/` root.
4. Update `scripts/gitops-sync.sh` and `environments/staging/setup-staging.yml` to point to the new centralized roles.
